terraform-azure-sql-database

Executive Summary

This Terraform module provisions a complete Azure SQL Database infrastructure including SQL Server, databases with flexible SKUs, elastic pools for resource sharing, failover groups for high availability, comprehensive security (auditing, threat detection, vulnerability assessment), network controls (firewall rules, VNet rules, private endpoints), Azure AD integration, and transparent data encryption with optional customer-managed keys. It supports the full lifecycle of enterprise relational database deployments on Azure.

Component Breakdown

SQL Server

MSSQL Server (v12.0 or v2.0) with configurable admin credentials, TLS 1.2, connection policy (Default/Proxy/Redirect), public network access control, outbound network restrictions, managed identity, and TDE with optional Key Vault CMK.

Databases

Flexible databases with configurable SKU, collation, max size, zone redundancy, read scale, auto-pause (serverless), geo-backup, storage account type, ledger support, and TDE. Supports short-term and long-term retention policies plus per-database threat detection.

Elastic Pools

Resource pools with configurable SKU (name, tier, family, capacity), per-database min/max capacity, max size, zone redundancy, license type, and maintenance configuration.

Failover Groups

High availability with automatic or manual failover policies. Configurable grace period for automatic failover. Read-only endpoint failover support. Maps databases to partner servers.

Security Stack

Server-level auditing policy (storage + Log Analytics), security alert policy with email notifications, and vulnerability assessment with recurring scans. Database-level threat detection with configurable disabled alerts.

Network Security

Firewall rules for IP-based access, VNet service endpoint rules for subnet-based access, and private endpoints with DNS zone groups for fully private connectivity.

Data Flow

  1. Clients connect via FQDN, routed through connection policy (Default/Proxy/Redirect)
  2. Firewall rules, VNet rules, and private endpoints control network access
  3. Azure AD or SQL authentication validates identity
  4. Queries are routed to databases (standalone or in elastic pools)
  5. Failover groups provide automatic redirection to partner server on failure
  6. TDE encrypts data at rest with Microsoft-managed or customer-managed keys
  7. Auditing policy logs operations to storage and/or Log Analytics
  8. Threat detection monitors for SQL injection, anomalous access, and brute force
  9. Vulnerability assessment runs recurring scans and reports findings

Industry Adaptation

Financial Services

Enable ledger databases for tamper-proof audit trails. CMK encryption, private endpoints only, full auditing, failover groups with automatic failover, vulnerability assessment with email alerts.

Healthcare

AAD-only authentication, TDE with CMK, 365-day audit retention, threat detection with admin email alerts, private endpoints with DNS zones, geo-backup enabled.

SaaS Platforms

Elastic pools for multi-tenant database consolidation, per-database min/max capacity, auto-pause for serverless workloads, read replicas for reporting.

Government

FedRAMP-aligned: CMK encryption, disabled public access, VNet rules, full auditing + vulnerability assessment, 730-day retention, zone-redundant databases.

Production Readiness Checklist

  • Configure Azure AD administrator with AAD-only authentication
  • Enable TDE with customer-managed keys in Key Vault
  • Set minimum TLS version to 1.2
  • Disable public network access, use private endpoints
  • Configure firewall rules for necessary IP ranges only
  • Set up failover groups for business-critical databases
  • Enable server-level auditing with appropriate retention
  • Configure security alert policy with admin email notifications
  • Enable vulnerability assessment with recurring scans
  • Configure short-term and long-term retention policies
  • Set up elastic pools for multi-database workloads
  • Apply resource tags for cost management

terraform apply tfplan

Prerequisites

  • Terraform >= 1.3.0
  • AzureRM provider >= 3.80.0
  • Existing Azure Resource Group
  • For failover groups: partner SQL Server in a different region
  • For CMK: Key Vault with appropriate access policies
